2020-2021

At Hotel Engine, me and four other new hire frontend developers were tasked with rebooting a stagnant UI which was a hodgepodge of Angular, HTML templates, and React. We systematically replaced old pages with best practices React and Typescript, added tests, and broke out common, reusable components in coordination with designers. I worked extensively on revamping the checkout process and making the interface modern and slick. Later, on the back office billing team, I worked to build a beautiful, responsive dashboard with styled components, while making the billing pages more performant as well.

2019

As a contractor for Aetna, I worked on the client side of a cross platform web app for finding doctors and medications, booking appointments, storing and viewing insurance info, including for dependents. My team worked extensively with React, Webpack, Redux, as well as testing tools like Karma and Mocha.

2017 - 2018

Missing home, I came back to rejoin my old pals at Rally. It was more of the same - working with React to build beautiful features and a component library to be used by the rest of the engineers. This is also when I took a keen interest in the performance of web applications, learning the Chrome profiling and debugging tools very well. I used those skills to make complicated, reusable components work well in a variety of use cases.

2015 - 2016

In fall of 2015, I was yearning for startup life. I found InfluxDB and was excited by the team, the vision and the ecosystem they were building. It didn't hurt that they were located in beautiful San Francisco. I joined the team to help build Chronograf, a graphing and dashboarding tool to visualize data stored in InfluxDB. We used the standard suite of React.js tools (i.e. redux, webpack) and a time-series graphing tool. I was a mentor for a couple of the newer engineers, and I enjoyed that responsibility.

waffle.io

This is waffle.io, a light-weight project management tool for teams of developers. We started waffle with the vision of making open source projects run more smoothly, and letting possible contributors figure how to most effectively help. It is built with a mongodb/node.js backend and a backbone.js/chaplin frontend.
